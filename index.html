Rotation Is All You Need

How rotary position embeddings turn token order into geometry

Step 1

Why Position?

Transformers need position information because self-attention alone does not know token order.

Self-attention sees token content, but position must be injected separately.

Step 2

Rotate Q/K

RoPE encodes position by rotating query/key vectors instead of adding a position vector.

qp = R(pθ) q
x' = x cos(pθ) - y sin(pθ)
y' = x sin(pθ) + y cos(pθ)

Step 4

Multi-frequency RoPE

RoPE splits a high-dimensional query/key vector into 2D pairs. Each pair rotates like a small clock.

What is a 2D pair?

Example query/key vector:

q = [x1, x2, x3, x4, x5, x6, x7, x8]
Pair 1 = (x1, x2) Pair 2 = (x3, x4) Pair 3 = (x5, x6) Pair 4 = (x7, x8)

RoPE treats every two hidden dimensions as a point on a 2D plane, then rotates that point.

What is θi?

θi = rotation speed for pair i

anglei = position p × θi

θi does not directly store the position. It tells each pair how fast to rotate when the position changes.

Why different θ?

Large θ values rotate quickly and capture local order. Small θ values rotate slowly and help with long-range structure.

Fast clocks

Early pairs have larger θ, so they rotate quickly. This helps distinguish nearby positions and local word order.

Slow clocks

Later pairs have smaller θ, so they rotate slowly. This helps preserve long-range position patterns.

RoPE is a positional embedding because position p changes the query/key vector. Instead of adding a position vector, RoPE rotates each 2D pair by p × θi.
