Transformer positional encodings

When attention loses order, position has to become geometry.

This demo uses the sentence from lecture, "I like black coffee", to compare position-vector methods and then trains a tiny attention subproblem to show why relative position methods like RoPE generalize better.

Part 1

Different ways to turn position into a vector.

Self-attention compares tokens as a set unless we inject order. The common move is to compute a position vector and combine it with the token embedding. Select a method and token to see the actual numbers used for that position.

Part 2

RoPE uses rotations instead of adding a new vector.

RoPE groups hidden dimensions into 2D pairs. At position m, pair i rotates by m * thetai. The dot product between query position m and key position n then depends on m - n.

[x,y] -> [x cos(m theta) - y sin(m theta), x sin(m theta) + y cos(m theta)]

Part 3

A tiny transformer attention training task.

The toy model is one attention scorer. For each query position m, it must put the most attention on the key at m - 2. Use the sliders to train on short sequences and test on longer sequences.

This isolates the positional part of a transformer: can the position representation learn a relative rule and extrapolate beyond training lengths?

Core takeaway

Absolute position encodings can tell a model where a token is. RoPE changes query/key geometry so attention scores naturally depend on how far two tokens are from each other, which is the pattern many sequence tasks actually need.
